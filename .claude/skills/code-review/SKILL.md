리뷰 대상: $ARGUMENTS

위 대상에 대해 전문적인 코드 리뷰를 수행하라.
리뷰 대상이 비어있으면 git diff를 통해 최근 변경된 파일을 자동으로 파악하여 리뷰하라.

## 실행 순서

### 1단계: 리뷰 대상 파악
- `$ARGUMENTS`가 있으면 해당 파일 경로 또는 기능 설명을 리뷰 대상으로 설정
- `$ARGUMENTS`가 없으면 `git diff HEAD~1` 또는 `git status`로 최근 변경 파일 자동 파악

### 2단계: 코드 읽기
리뷰 대상 파일을 모두 읽고 관련 컨텍스트(타입 정의, 의존 모듈 등)도 함께 파악하라.

### 3단계: 5가지 핵심 영역 검토
다음 순서로 철저히 검토하라:

1. **코드 품질 및 가독성** — 네이밍, SRP, DRY, 복잡도, 주석, 매직 넘버
2. **보안** — 입력값 검증, SQL 인젝션/XSS/CSRF, 민감 데이터 처리, 인증/인가
3. **성능** — N+1 쿼리, 메모리 누수, 비효율 알고리즘, 캐싱 기회, 불필요한 재렌더링
4. **유지보수성 및 확장성** — SOLID 원칙, 에러 처리, 로깅, 하드코딩 여부
5. **테스트** — 커버리지, 엣지 케이스, 테스트 독립성

### 4단계: 리뷰 결과 출력
다음 형식으로 한국어 리뷰 결과를 작성하라:

```
## 📋 코드 리뷰 요약
[전반적인 코드 품질에 대한 2-3문장 요약]

## 🚨 Critical (즉시 수정 필요)
- **위치**: 파일명:라인번호
- **문제**: 설명
- **이유**: 왜 문제인지
- **해결책**: 구체적인 수정 방법

## ⚠️ Major (수정 권장)
[동일 형식]

## 💡 Minor (개선 제안)
[동일 형식]

## ✅ 잘된 점
[긍정적인 부분, 좋은 패턴]

## 📝 총평
[종합적인 평가 및 다음 단계 제안]
```

이슈가 없는 섹션은 "없음"으로 표시하라.
